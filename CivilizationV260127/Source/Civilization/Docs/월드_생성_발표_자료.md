# 런타임 월드 생성 로직 — 발표 자료 (약 5분)

## 개요

**WorldSettingMenu**에서 만들 월드 설정을 정한 뒤, "월드 생성"을 실행하면 **Loading 레벨**을 거쳐 **런타임 중에** 맵 데이터가 만들어지고, **InGame 레벨**에서 타일/도시 액터가 차례로 스폰됩니다.  
이 문서는 그 과정을 **주요 함수** 중심으로 정리한 발표용 요약입니다.

---

## 1. 전체 흐름 요약

```
[WorldSettingMenu]  설정 입력 → GameInstance에 저장 → "시작" 버튼
        ↓
[SuperGameInstance::OpenLevel(InGame)]  → TargetLevel=InGame 저장, Loading 레벨 오픈
        ↓
[Loading 레벨]  LoadingUI 표시
        ↓
[LoadingUI]  월드 데이터 생성(0%→50%) → 레벨 비동기 로드(50%→70%) → 70% 도달 시 InGame으로 전환
        ↓
[InGame 레벨]  LoadingTilesUI 표시, WorldSpawner 동작
        ↓
[LoadingTilesUI + WorldSpawner]  타일/도시 액터 스폰(70%→90%) → 정리(90%→100%) → 위젯 종료
```

- **데이터 생성**: Loading 단계에서 `UWorldComponent`가 맵 **로그릭/데이터**만 만듦(타일 데이터, 지형, 자원, 도시 좌표 등).
- **액터 스폰**: InGame 단계에서 `AWorldSpawner`가 그 데이터를 바탕으로 `AWorldTileActor`, `ACityActor` 등을 **비동기로** 스폰.

---

## 2. WorldSettingMenu → 설정 저장

| 역할 | 위치 | 설명 |
|------|------|------|
| 설정 저장 | `WorldSettingMenuUI` | 슬라이더/콤보 등에서 `FWorldConfig` 수집 후 `GameInstance->SetWorldConfig(NewWorldConfig)` 호출 |
| 국가 이름 | `WorldSettingMenuUI` | 선택된 국가 이름 배열을 `GameInstance->SetCountryNames(...)`로 저장 |
| 진행 트리거 | `WorldSettingMenuUI` | "게임 시작" 시 `GameInstance->OpenLevel(InGameLevel)` 호출 |

**SuperGameInstance**에서 실제로 갖는 값:

- `CurrentWorldConfig` — 맵 반지름, 플레이어 수, 바다 비율, 평지/숲/사막/툰드라 비율 등
- `CountryNames` — 플레이어/AI용 국가 이름 (최대 8개)

## 3. SuperGameInstance — 설정·월드·레벨 중계

| 함수/멤버 | 파일 | 역할 |
|------------|------|------|
| `SetWorldConfig(const FWorldConfig&)` | SuperGameInstance.cpp | 월드 설정 저장 (`CurrentWorldConfig = NewSettings`) |
| `GetWorldConfig()` | SuperGameInstance.h | 로딩/월드 생성 시 사용할 설정 반환 |
| `OpenLevel(TSoftObjectPtr<UWorld> newLevel)` | SuperGameInstance.cpp | `TargetLevel = newLevel` 저장 후 **Loading** 레벨로 이동 (`UGameplayStatics::OpenLevel(this, "Loading")`) |
| `GetTargetLevelPackageName()` | SuperGameInstance.h | 나중에 InGame 레벨 비동기 로드할 때 쓸 패키지 경로 반환 |
| `SetGeneratedWorldComponent(UWorldComponent*)` | SuperGameInstance.cpp | Loading에서 만든 `UWorldComponent`를 저장 → InGame/WorldSpawner에서 사용 |
| `GetGeneratedWorldComponent()` | SuperGameInstance.h | WorldSpawner·매니저들이 월드 데이터에 접근할 때 사용 |

즉, **설정·타겟 레벨·생성된 월드 컴포넌트**를 한 군데서 관리하는 역할입니다.

## 4. Loading 레벨 & LoadingUI (0% → 70%)

### 4.1 진입 시퀀스

| 단계 | 함수 | 파일 | 설명 |
|------|------|------|------|
| 1 | `NativeConstruct()` | LoadingUI.cpp | 위젯 생성 시 한 번 호출. `GenerateWorldAsync()` 호출로 월드 생성 시작 |
| 2 | `GenerateWorldAsync()` | LoadingUI.cpp | `GameInst->ClearGeneratedWorldComponent()` 후 새 `UWorldComponent` 생성, `GameInst->GetWorldConfig()`로 설정 주입, `OnWorldGenerated` 바인딩. **새 게임**이면 `WorldComponent->GenerateWorld()`, **로드**면 `GenerateWorldFromSaveData(...)` 호출 |
| 3 | `OnWorldGenerated(bool)` | LoadingUI.cpp | 월드(데이터) 생성 완료 시 호출. `GameInst->SetGeneratedWorldComponent(WorldComponent)`로 저장, `LoadLevelAsync()`로 InGame 레벨 비동기 로드 시작 |
| 4 | `LoadLevelAsync()` | LoadingUI.cpp | `GameInst->GetTargetLevelPackageName()`으로 InGame 패키지 경로 가져와 `LoadPackageAsync(..., OnLevelLoaded)` 호출 |
| 5 | `NativeTick(...)` | LoadingUI.cpp | `curPercent`를 `targetPercent`까지 보간해 진행률 표시. **70% 도달** 시 `UGameplayStatics::OpenLevel(this, TargetLevelName)`으로 **InGame** 레벨로 전환 |

### 4.2 진행률 의미

- **0% → 50%**: 월드 **데이터** 생성 중 (`UWorldComponent::GenerateWorld()` 또는 `GenerateWorldFromSaveData()`).
- **50% → 70%**: InGame 레벨 **비동기 로드** 중.
- **70% 도달**: InGame 레벨로 최종 전환.

## 5. WorldComponent — 월드 데이터 생성 (로그릭만)

타일/지형/자원/도시 위치 등 **맵 로직**을 담당합니다. **액터 스폰은 하지 않습니다.**

### 5.1 메인 진입점

| 함수 | 파일 | 역할 |
|------|------|------|
| `GenerateWorld()` | WorldComponent.cpp | 전체 월드 데이터 생성 파이프라인 진입. 실패 시 `bIsWorldGenerated = false` 유지 |
| `GenerateWorldFromSaveData(...)` | WorldComponent.cpp | 세이브 데이터로 타일·설정·도시 좌표만 복원 (스폰 없음) |

### 5.2 GenerateWorld() 내부 호출 순서

1. **ClearWorld()** — 기존 `HexTiles` 비우고 `bIsWorldGenerated = false`.
2. **InitializeHexTiles()** — `WorldConfig.WorldRadius` 기준 원형 육각 그리드에 대해 `CreateTileAtHex(HexPos)` 호출해 **빈 타일(UWorldTile)** 생성.
3. **GenerateTerrain()** → **GeneratePangaeaTerrain()** — 바다 비율에 맞춰 육지/바다 배치 (판게아 스타일).
4. **GenerateCities(NumCities)** — 육지 후보 중 “이웃 6칸 중 육지 수 ≥ 5, 서로 거리 ≥ MinHexDistance” 조건으로 **도시 좌표만** 선정해 `StartingCityHexes`에 저장. **액터는 생성하지 않음**.
5. **GenerateClimateZones()** — 육지 타일에 툰드라/사막/온대 비율 적용.
6. **GenerateLandTypes()** — 평지/언덕/산 비율 적용.
7. **GenerateForests()** — 숲 비율 적용 (도시 타일 제외).
8. **GenerateResources()** — 사치 → 전략 → 보너스 순으로 자원 배치 (데이터테이블·호환 조건 사용).
9. **RecalculateAllTileYields()** — 식량/생산/골드/과학/신앙/이동비용/전투보너스 등 캐시 갱신.
10. **bIsWorldGenerated = true**, **OnWorldGenerated.Broadcast(true)** 로 LoadingUI에 “데이터 생성 완료” 통보.

### 5.3 자주 쓰는 헬퍼

| 함수 | 용도 |
|------|------|
| `GetTileAtHex(FVector2D)`, `CreateTileAtHex(FVector2D)` | Hex 좌표로 타일 조회/생성 |
| `GetHexesInRadius(CenterHex, Radius)` | 반경 내 Hex 좌표 배열 |
| `HexToWorld(Hex)`, `WorldToHex(World)` | Hex ↔ 월드 좌표 변환 |
| `GetStartingCityHexes()` | 도시 스폰 위치(스폰은 WorldSpawner 담당) |

## 6. InGame 레벨 & LoadingTilesUI (70% → 100%)

InGame 레벨에 들어오면 **LoadingTilesUI**가 70%부터 100%까지 진행률을 채우며, **WorldSpawner**에게 “타일·도시 스폰”과 “도시 배정”을 맡깁니다.

### 6.1 LoadingTilesUI

| 함수 | 파일 | 역할 |
|------|------|------|
| `NativeConstruct()` | LoadingTilesUI.cpp | 초기 진행률 70%, `StartTileSpawning()` 호출 |
| `StartTileSpawning()` | LoadingTilesUI.cpp | 월드 내 `AWorldSpawner` 검색, `OnTileSpawnCompleted` 바인딩 후 `WorldSpawner->SpawnAllTiles()`, `SpawnAllCities()`, `AssignCitiesToPlayers()` 호출 |
| `OnTileSpawnCompleted()` | LoadingTilesUI.cpp | 스폰 끝나면 호출. 메인메뉴 로드 모드일 때 `WorldSpawner->RestoreGameStateFromSave(PendingLoadData)` 실행, `bDataRestored = true` 등 설정 |
| `NativeTick(...)` | LoadingTilesUI.cpp | 70%→90%→100% 진행률 갱신. **100%**가 되면 `RemoveFromParent()`로 위젯 제거 |

즉, “타일/도시 스폰 + (로드 시) 게임 상태 복원”이 끝난 뒤 진행률을 100%까지 올리고, 그때 로딩 UI를 닫습니다.

## 7. WorldSpawner — 타일·도시 액터 스폰

BeginPlay에서 **GameInstance**로부터 `WorldComponent`와 매니저들(UnitManager, FacilityManager, BorderManager 등)을 준비하고, **LoadingTilesUI**가 호출하는 시점에 실제 스폰을 수행합니다.

### 7.1 준비 (BeginPlay)

| 동작 | 파일 | 설명 |
|------|------|------|
| WorldComponent 참조 | WorldSpawner.cpp | `WorldComponent = SuperGameInst->GetGeneratedWorldComponent()` |
| 매니저 생성/등록 | WorldSpawner.cpp | UnitManager, FacilityManager, BorderManager, DiplomacyManager, AIPlayerManager 생성 후 GameInstance에 Set. (스폰은 LoadingTilesUI에서 시작할 때까지 대기) |

### 7.2 타일 스폰

| 함수 | 파일 | 역할 |
|------|------|------|
| `SpawnAllTiles()` | WorldSpawner.cpp | `ClearAllTiles()` 후 `WorldComponent->GetAllTiles()`로 타일 목록 확보. `TilesToSpawn`·`CurrentSpawnIndex` 초기화하고 `ProcessAsyncSpawn`을 **타이머**(약 0.016초 간격)로 반복 호출 |
| `ProcessAsyncSpawn()` | WorldSpawner.cpp | 한 번에 `SpawnPerFrame`개(예: 50개)씩 `SpawnTileActor(TileData)` 호출. 다 스폰하면 타이머 해제 후 **OnTileSpawnCompleted.Broadcast()** |
| `SpawnTileActor(UWorldTile*)` | WorldSpawner.cpp | `TileActorClass`로 `AWorldTileActor` 스폰, `SetTileData(TileData)` 호출 후 `TileActors` 맵에 Hex–액터 쌍으로 저장 |

타일 스폰은 **프레임당 개수 제한**으로 나누어 해서 프리즈를 줄이는 비동기 스폰 구조입니다.

### 7.3 도시 스폰·배정

| 함수 | 파일 | 역할 |
|------|------|------|
| `SpawnAllCities()` | WorldSpawner.cpp | `WorldComponent->GetStartingCityHexes()`로 좌표만 받아, 각 Hex에 `SpawnCityActorAtHex(Hex)` 호출 |
| `SpawnCityActorAtHex(FVector2D Hex)` | WorldSpawner.cpp | `WorldComponent->HexToWorld(Hex)`로 위치 계산, 지형 높이에 따라 Z 오프셋 적용 후 `ACityActor` 스폰, `CityActors`에 등록 |
| `AssignCitiesToPlayers()` | WorldSpawner.cpp | `GetStartingCityHexes()` 순서대로 각 플레이어 인덱스에 맞는 `PlayerState`를 찾아, 도시 좌표·CityComponent·ResearchComponent 설정, 도시 반경 1칸 타일을 해당 플레이어 소유로 넣고, 국경선·AIPlayerManager 초기화, 마지막에 `MoveCameraToPlayerCity()`로 카메라를 플레이어 0 도시로 이동 |

### 7.4 로드 시 복원

| 함수 | 파일 | 역할 |
|------|------|------|
| `RestoreGameStateFromSave(const FGameSaveData&)` | WorldSpawner.cpp | 턴, 플레이어/도시 데이터, 시설, 외교, 유닛, 국경선, AI 등을 SaveLoadManager 등으로 복원. **월드 타일/도시 좌표는 이미 GenerateWorldFromSaveData로 반영된 상태**를 전제로 함 |

## 8. WorldTileActor — 타일 시각·상호작용

**WorldSpawner**가 스폰한 각 타일의 “보이는 부분”과 “클릭/호버”를 담당합니다.

| 함수/역할 | 파일 | 설명 |
|-----------|------|------|
| `SetTileData(UWorldTile*)` | WorldTileActor.cpp | 참조 타일 데이터 설정 후 `UpdateVisual()` 호출 |
| `UpdateVisual()` | WorldTileActor.cpp | 지형(육지/바다), 기후, 땅타입(평지/언덕/산), 숲, 자원에 따라 TileMesh/ForestMesh/ResourceMesh·스케일·오프셋 설정. 데이터테이블에서 메시 조회 |
| 클릭/호버 | WorldTileActor.cpp | `OnTileClicked`, `OnBeginCursorOver`, `OnEndCursorOver`에서 도시·건설자·일반 타일 구분, UnitManager 이동/전투 선택 등에 이벤트 전달 |

데이터는 전부 `UWorldTile* TileData`와 `UWorldComponent`에 있고, **WorldTileActor**는 그걸 “한 칸” 단위로 보여주고 입력을 넘기는 역할입니다.

## 9. 5분 발표용 순서 제안

1. **도입 (30초)**  
   “메뉴에서 월드 옵션을 정하고, 게임을 시작하면 **Loading → InGame** 두 단계로 나뉘어, **데이터 생성**과 **액터 스폰**이 순서대로 이뤄집니다.”

2. **설정 저장 (30초)**  
   “WorldSettingMenu에서 `SetWorldConfig` / `SetCountryNames`로 GameInstance에 저장하고, 시작 시 `OpenLevel(InGame)`으로 TargetLevel을 잡은 뒤 **Loading** 레벨로 들어갑니다.”

3. **Loading & 월드 데이터 (1분 30초)**  
   “LoadingUI가 `GenerateWorldAsync()`에서 **UWorldComponent**를 만들고, 설정을 넣은 뒤 `GenerateWorld()`를 호출합니다.  
   `GenerateWorld()`는 ClearWorld → InitializeHexTiles → GeneratePangaeaTerrain → GenerateCities(도시 좌표만) → 기후/지형/숲/자원 → RecalculateAllTileYields 순입니다.  
   끝나면 `OnWorldGenerated`로 GameInstance에 WorldComponent를 넘기고, 이어서 InGame 레벨을 비동기 로드하다가 70%가 되면 실제로 InGame으로 넘어갑니다.”

4. **InGame & 타일/도시 스폰 (1분 30초)**  
   “InGame에는 **LoadingTilesUI**와 **WorldSpawner**가 있습니다.  
   LoadingTilesUI가 `SpawnAllTiles()`, `SpawnAllCities()`, `AssignCitiesToPlayers()`를 호출하고, WorldSpawner는 `ProcessAsyncSpawn()`으로 프레임당 N개씩 타일 액터를 스폰합니다.  
   스폰이 끝나면 `OnTileSpawnCompleted`가 호출되고, 로드 모드면 `RestoreGameStateFromSave`로 게임 상태를 복원합니다.  
   각 타일 액터는 `SetTileData`로 데이터를 받고 `UpdateVisual()`로 지형/기후/자원에 맞는 메시를 보여줍니다.”

5. **마무리 (30초)**  
   “정리하면, **설정은 GameInstance**, **맵 로직은 WorldComponent**, **스폰은 WorldSpawner**, **한 칸의 표현은 WorldTileActor**가 담당하고, Loading ↔ InGame 두 레벨과 두 로딩 UI가 이 흐름을 이어주는 구조입니다.”

## 10. 클래스·파일 요약표

| 클래스/컴포넌트 | 파일 | 담당 |
|-----------------|------|------|
| USuperGameInstance | SuperGameInstance.h/cpp | 설정·타겟 레벨·GeneratedWorldComponent 보관 |
| ULoadingUI | LoadingUI.h/cpp | 0–70% 진행, 월드 데이터 생성 트리거·레벨 전환 |
| UWorldComponent | WorldComponent.h/cpp | 육각 그리드·지형·자원·도시 좌표 등 **데이터만** 생성 |

| AWorldSpawner | WorldSpawner.h/cpp | 타일/도시 액터 비동기 스폰, 도시–플레이어 배정, 로드 시 게임 상태 복원 |
